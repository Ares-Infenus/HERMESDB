<b>Componente: data_config.csv</b><br/><b>Descripción</b><br/>Este componente es un archivo CSV que almacena las credenciales necesarias para acceder a las instancias de MetaTrader 5 (MT5) en diferentes servidores. Las credenciales consisten en los nombres de usuario y contraseñas asociados a cada servidor, utilizados para autenticar las conexiones en una cuenta demo del sistema de trading automatizado.<br/><br/><b>Propósito</b><br/>Este archivo proporciona una fuente centralizada para la autenticación en el entorno de trading. Se utiliza durante la configuración inicial o la integración con los servidores de MT5 en un entorno de prueba.<br/><br/><b>Ubicación en la Arquitectura</b><br/>Este componente es parte del proceso de autenticación dentro de la arquitectura del sistema de trading. Debe ser cargado al sistema en las etapas iniciales del proceso de configuración y de despliegue en entornos de prueba.<br/><br/><b>Proceso de Integración</b><br/>Carga: Durante la configuración inicial, el archivo debe ser cargado de manera directa y sin restricciones de seguridad estrictas, dado que se trata de una cuenta demo.<br/>Consumo por el Sistema de Autenticación: El archivo es consumido por el componente de autenticación encargado de gestionar las conexiones a MT5.<br/><br/><b>Mantenimiento y Recomendaciones</b><br/>Actualización de Credenciales: Si es necesario, se pueden cambiar las credenciales de la cuenta demo en cualquier momento, dado que no hay riesgos de seguridad críticos asociados.<br/>Verificación de Integridad: Se recomienda verificar la integridad del archivo antes de cargarlo para asegurar que las credenciales sean correctas.<br/><br/><b>Impacto en el Sistema</b><br/>El manejo adecuado de este archivo es esencial para el funcionamiento del sistema de trading automatizado en el entorno de prueba. Cualquier error en las credenciales puede afectar la capacidad de conexión a los servidores de MT5.<br/>
